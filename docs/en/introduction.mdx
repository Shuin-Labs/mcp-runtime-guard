## Overview

`mcp-runtime-guard` is a transparent CLI proxy that sits between AI agents (Claude Code, Cursor, etc.) and real MCP servers.

It blocks tool calls that violate YAML-defined policies and records all operations in an audit log.

```
Claude Code / Cursor
    │ stdio (JSON-RPC 2.0)
    ▼
mcp-runtime-guard  ← policy enforcement here
    │ stdio (JSON-RPC 2.0)
    ▼
Real MCP Server
```

## Why you need it

MCP servers have access to the filesystem, shell, and external APIs. AI agents may accidentally — or through malicious instructions — perform operations like:

- Reading credential files (`.env`, `~/.ssh/id_rsa`)
- Executing shell commands
- Making HTTP requests to unintended external services

`mcp-runtime-guard` prevents these with policies.
